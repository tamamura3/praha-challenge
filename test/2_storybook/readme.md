# 課題3
Storybookのメリット
- アプリを動かさずにUIコンポーネントの作成や変更、確認ができる
- UIコンポーネントの管理ができるようになる
Storybookを使う＝UIのドキュメントを作ることになる
仕様把握、類似のコンポーネント作成の防止などに役立つ。
Storybookのデメリット
- アプリとStorybookの両方を管理する必要がある
